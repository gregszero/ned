# Separate Ned repo with OpenFang as upstream

## Context
Ned currently clones the OpenFang repo directly. Agent-generated files on Ned (skills, pages, workspace data) will conflict with pulls from master. We want Ned to have its own GitHub repo (`gregszero/ned-openfang`) with the OpenFang repo as an upstream remote for pulling code updates.

## Steps

### 1. Create `gregszero/ned-openfang` repo on GitHub
```bash
gh repo create gregszero/ned-openfang --private --source /home/ned/openfang
```

### 2. Reconfigure remotes on Ned
On Ned's existing clone:
```bash
cd /home/ned/openfang
git remote rename origin upstream    # openfang repo becomes "upstream"
git remote add origin git@github.com:gregszero/ned-openfang.git
git push -u origin master            # push current state to ned's repo
```

Now: `origin` = Ned's repo, `upstream` = OpenFang source.

### 3. Update `bin/deploy`
The deploy script changes to:
- Push `master` to **upstream** (the OpenFang repo) from local dev
- On Ned: fetch upstream, merge `upstream/master`, push to origin (backup), bundle, migrate, restart

```bash
# Local:
git push upstream master   # or just "origin" from dev machine

# On Ned:
git fetch upstream
git merge upstream/master --no-edit
git push origin master     # backup to ned-openfang
bundle install --quiet
./openfang.rb db_migrate
systemctl --user restart openfang
```

### 4. Update local dev remote (optional)
Dev machine keeps `origin` pointing to OpenFang. No change needed — deploy script already pushes to `origin` which is OpenFang.
